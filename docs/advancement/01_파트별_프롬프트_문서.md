# 파트별 프롬프트 문서

> **목적**: 서비스 고도화 기획자료를 **파트별로** 진행할 때, 복사·붙여넣기해서 쓸 수 있는 **지시용 프롬프트**를 정리한다.  
> **사용법**: 아래 해당 파트의 "기본 프롬프트" 또는 "세부 프롬프트"를 복사해 AI/에이전트에 붙여넣고, 지시된 파일만 열어서 실행한다. 진행 순서는 Part A → B → C → D → 부록이다.  
> **맥락 유지**: 이전 파트를 마친 뒤에는 해당 파트 문서의 **"핵심 산출 요약"** 섹션을 다음 파트 프롬프트에 포함하거나, "다음 파트 실행 전 Part A 문서의 'Part A 핵심 산출 요약'을 읽고 그 내용을 전제로 Part B를 진행해라"처럼 지시하면 된다.

## Part A용 프롬프트

### A-1. 기본 프롬프트 (Part A 전체 진행)

```
다음 작업을 수행해줘.

[목표]
서비스 고도화 기획자료의 Part A(서비스 개요 및 현재 기능)를 작성·갱신한다.

[참조할 문서]
- docs/development_progress.md (완료 기능·파일 목록)
- docs/zodiac_service_prd.md (기능 우선순위·MVP/Phase 2·3 기능)
- 프로젝트 실제 구조: src/app 아래 페이지·API 경로 (development_progress와 다르면 실제 구조 기준으로 정리)

[작업할 문서]
- docs/advancement/part_a_서비스_개요_및_현재_기능.md

[작업 내용]
1. A.1 현재 구현 기능 (As-Is)
   - 기능 목록 표: 영역, 기능, 상세, 구현 상태. (운세, 별자리, 궁합, 출생 차트, 클라이언트 상태 등 development_progress와 src/app 구조를 반영해 정확히 채운다.)
   - 미구현(PRD 명시) 항목 표: 기능명, PRD 우선순위, 비고. (인증, 푸시, 소셜, AI 개인화, 커뮤니티, 전문가 상담 등)
2. A.2 개념 정의
   - 지표·사업 용어: MAU, DAU, ARPU, 리텐션 D7/D30, 유료 전환율
   - 서비스·점성술 용어: Big Three, 상승궁, 트랜짓, Freemium, 구독/단건 결제
   - 서비스 핵심 개념: 현재(템플릿/규칙 기반) → 고도화(트랜짓·AI 개인화) 한 문단
3. 문서 끝의 "Part A 핵심 산출 요약" 섹션을 갱신한다. (구현 완료 목록, 미구현 목록, 용어·개념 한 줄 요약)

현재 앱은 locale 라우팅 없이 페이지가 구성되어 있으면 그 사실을 주석이나 참고에 반영해줘.
```

### A-2. 세부 프롬프트 (기능 목록만 갱신)

```
docs/advancement/part_a_서비스_개요_및_현재_기능.md의 A.1 "기능 목록" 표와 "미구현 항목" 표만 갱신해줘. docs/development_progress.md와 src/app 디렉터리 구조를 기준으로, 실제 구현된 기능과 PRD에 있는 미구현 기능을 반영해줘.
```

### A-3. 세부 프롬프트 (개념 정의만 갱신)

```
docs/advancement/part_a_서비스_개요_및_현재_기능.md의 A.2 "개념 정의"와 "서비스 핵심 개념"만 검토·갱신해줘. docs/zodiac_service_prd.md 부록의 용어 정의와 맞추고, 고도화 방향(트랜짓·AI)이 한 문단에 명확히 드러나게 해줘.
```

## Part B용 프롬프트

### B-1. 기본 프롬프트 (Part B 전체 진행, Part A 맥락 이어짐)

```
다음 작업을 수행해줘.

[전제 맥락]
Part A가 완료된 상태다. docs/advancement/part_a_서비스_개요_및_현재_기능.md의 "Part A 핵심 산출 요약" 섹션을 먼저 읽고, 그 내용(구현 완료·미구현·용어·개념)을 전제로 Part B를 진행해줘.

[목표]
서비스 고도화 기획자료의 Part B(고도화 방향 및 로드맵)를 작성·갱신한다.

[참조할 문서]
- docs/advancement/part_a_서비스_개요_및_현재_기능.md (특히 Part A 핵심 산출 요약)
- docs/zodiac_service_prd.md (Phase 2·3, 로드맵)
- docs/development_progress.md ("다음 단계" 섹션)

[작업할 문서]
- docs/advancement/part_b_고도화_방향_및_로드맵.md

[작업 내용]
1. B.1 기능 고도화 우선순위
   - 표: 기능, 비즈니스 임팩트, 기술 난이도, 선행 조건, 목표 시기(가정), 담당. Part A 미구현 항목 + 결제/구독을 포함해 PRD Phase 2·3와 맞춰 채운다.
   - 고도화 개념 요약 표: 축(콘텐츠/참여/수익), 현재(Part A 기준), 고도화 방향
2. B.2 비기능 고도화
   - 표: 영역, 항목, 방향. (성능·캐시·API, 보안·개인정보·결제, 확장성·DB)
3. B.3 로드맵
   - 기존 mermaid 다이어그램 유지. 단계별 핵심 배달물 표(Phase 1·2·3, 기간·배달물)를 PRD와 맞게 갱신
4. 문서 끝의 "Part B 핵심 산출 요약" 섹션을 갱신한다. (우선순위 요약, 고도화 3축, 비기능 요약, 로드맵 요약)
```

### B-2. 세부 프롬프트 (우선순위 표만 갱신)

```
docs/advancement/part_b_고도화_방향_및_로드맵.md의 B.1 "기능 고도화 우선순위" 표만 갱신해줘. part_a의 미구현 항목과 docs/zodiac_service_prd.md Phase 2·3를 반영해, 선행 조건·목표 시기(가정)·담당을 채워줘.
```

## Part C용 프롬프트

### C-1. 기본 프롬프트 (Part C 전체 진행, Part A·B 맥락 이어짐)

```
다음 작업을 수행해줘.

[전제 맥락]
Part A·B가 완료된 상태다. docs/advancement/part_a_서비스_개요_및_현재_기능.md의 "Part A 핵심 산출 요약", docs/advancement/part_b_고도화_방향_및_로드맵.md의 "Part B 핵심 산출 요약"을 먼저 읽고, 그 내용을 전제로 Part C를 진행해줘.

[목표]
서비스 고도화 기획자료의 Part C(IPO/투자자 관점 준비사항)를 작성·갱신한다.

[참조할 문서]
- part_a, part_b (각 핵심 산출 요약)
- docs/zodiac_service_prd.md (시장 2장, 타겟 3장, 수익 5장, 리스크 8장, KPI 9장)
- docs/competitor_analysis.md (차별화·경쟁 포지셔닝 요약)

[작업할 문서]
- docs/advancement/part_c_IPO_투자자_준비사항.md

[작업 내용]
1. C.1 기획 대표 준비사항
   - 시장·사업성: 시장 규모·성장률, 타겟·페르소나 요약, 수익 모델 요약, 경쟁 포지셔닝(차별화 3가지)
   - 지표·로드맵: KPI 대시보드 요약, 12–24개월 로드맵(Part B와 연계)
   - 리스크·규제: 리스크 매트릭스 요약, 규제 체크리스트
2. C.2 CTO 준비사항
   - 기술 스택·아키텍처(현재·12개월 계획), 확장성·안정성, 보안·컴플라이언스, 팀·일정
3. C.3 공통
   - "현재 상태 한 장 요약" 표: Part A 반영
   - "다음 12개월 핵심 마일스톤" 5~7개: Part B 로드맵과 연계
4. 문서 끝의 "Part C 핵심 산출 요약"을 갱신한다.
```

### C-2. 세부 프롬프트 (기획 대표만)

```
docs/advancement/part_c_IPO_투자자_준비사항.md의 C.1 섹션만 갱신해줘. docs/zodiac_service_prd.md 2·3·5·8·9장과 docs/competitor_analysis.md를 참조해, 시장·수익·차별화·KPI·로드맵·리스크·규제를 투자자 메모용으로 정리해줘.
```

### C-3. 세부 프롬프트 (CTO만)

```
docs/advancement/part_c_IPO_투자자_준비사항.md의 C.2 섹션만 갱신해줘. 현재 스택(Next.js 15, Cloudflare 등)과 12개월 내 DB·인증·결제·캐시 계획, 확장성·보안·팀 구성을 투자자 질의응답용으로 2~4페이지 분량 요약 수준으로 써줘.
```

## Part D용 프롬프트

### D-1. 기본 프롬프트 (Part D 전체 진행, Part B·C 맥락 이어짐)

```
다음 작업을 수행해줘.

[전제 맥락]
Part B·C가 완료된 상태다. docs/advancement/part_b_고도화_방향_및_로드맵.md의 "Part B 핵심 산출 요약", docs/advancement/part_c_IPO_투자자_준비사항.md의 "Part C 핵심 산출 요약"(특히 CTO 기술·확장성)을 참조해 Part D를 진행해줘.

[목표]
서비스 고도화 기획자료의 Part D(서비스 확장 준비)를 작성·갱신한다. Part C가 "무엇을 준비할지"라면, Part D는 "어떻게 확장할지(실행)"를 구체화한다.

[참조할 문서]
- part_b (비기능 고도화·DB 도입 시점)
- part_c (CTO 확장성·운영)

[작업할 문서]
- docs/advancement/part_d_서비스_확장_준비.md

[작업 내용]
1. D.1 트래픽·데이터 확장: 캐시 전략(Cloudflare), DB·스토리지 도입 시점, 세션·파일 스토리지
2. D.2 다국어·현지화: 현재 5개 언어, 신규 언어 추가·현지화 프로세스·체크리스트
3. D.3 수익화 연동: 결제(PG·빌링·환불), 광고 시 고려사항
4. D.4 모니터링·운영: 에러·성능·비용, 배포·롤백·장애 대응
5. 문서 끝의 "Part D 핵심 산출 요약"을 갱신한다.
```

### D-2. 세부 프롬프트 (섹션별)

```
docs/advancement/part_d_서비스_확장_준비.md에서 다음 섹션만 갱신해줘.

- D.1만: 트래픽·캐시·DB 도입 시점·세션·파일 스토리지를 실행 관점으로 구체화해줘. part_b의 비기능 고도화(확장성)와 맞춰줘.
- D.2만: 다국어 5개 유지, 신규 언어 추가 절차, 현지화(통화·날짜·라우팅) 체크리스트를 추가해줘.
- D.3만: 결제 파이프라인·환불 정책, 광고 연동 시 브랜드·개인정보 고려사항을 추가해줘.
- D.4만: 에러 추적·성능·비용 대시보드, 배포·롤백·장애 에스컬레이션을 정리해줘.
```

## 통합본 갱신용 프롬프트

### 통합-1. Part A–D·부록을 통합본에 반영

```
docs/advancement/ 폴더의 Part A, B, C, D, 부록 문서 내용을 docs/service_advancement_plan.md 한 문서에 순서대로 반영해줘. 각 파트 문서의 "핵심 산출 요약"은 통합본에는 넣지 말고, 본문만 Part A → Part B → Part C → Part D → 부록 순으로 병합해줘. 상대 경로(../)는 통합본 위치에 맞게 docs/ 기준으로 수정해줘.
```
