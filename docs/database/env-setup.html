Cloudflare Workers 환경변수 설정

.env.local은 로컬 개발용입니다. 배포된 Worker에는 별도로 환경변수를 설정해야 합니다.

ℹ️ Secret 명령어는 값을 암호화하여 저장합니다. 터미널에서 값을 입력 프롬프트로 받습니다.
Terminalbash
# Supabase URL (plain var)
npx wrangler secret put NEXT_PUBLIC_SUPABASE_URL

# Supabase Anon Key
npx wrangler secret put NEXT_PUBLIC_SUPABASE_ANON_KEY

# Supabase Service Role Key (⚠️ 가장 민감한 키)
npx wrangler secret put SUPABASE_SERVICE_ROLE_KEY

# (선택) AdSense
npx wrangler secret put NEXT_PUBLIC_ADSENSE_PUBLISHER_ID

# 설정된 secret 목록 확인
npx wrangler secret list
1

Cloudflare Dashboard 접속

dash.cloudflare.com → Workers & Pages

2

Worker 선택

Workers & Pages 목록에서 star Worker 클릭

3

Settings → Variables and Secrets

상단 탭에서 Settings → Variables and Secrets 메뉴 이동

4

환경변수 추가

Add variable 클릭 → 키/값 입력 → Encrypt 체크박스로 Secret 설정

⚠️ SUPABASE_SERVICE_ROLE_KEY는 반드시 Encrypt(암호화) 체크
5

Save and Deploy

저장 후 Worker가 자동으로 재배포됩니다.

🚫 wrangler.toml에 실제 키 값을 직접 입력하지 마세요. 이 파일은 git에 커밋되므로 보안 사고의 원인이 됩니다. 플레이스홀더 주석만 포함되어 있으며 실제 값은 절대 입력하지 않습니다.
wrangler.toml현재 상태 (안전)
# 환경 변수 (프로덕션)
# 실제 값은 Cloudflare Dashboard에서 설정하거나 wrangler secret put 명령으로 설정
# [vars]
# NEXT_PUBLIC_SUPABASE_URL = "https://xxx.supabase.co"
# NEXT_PUBLIC_SUPABASE_ANON_KEY = "xxx"
# SUPABASE_SERVICE_ROLE_KEY = "xxx"  (wrangler secret put SUPABASE_SERVICE_ROLE_KEY 권장)

템플릿 데이터 Supabase 업로드

Terminal
