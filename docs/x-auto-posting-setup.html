API 키 4개 발급

⏱ 약 3분
🔑 키 발급 경로
Developer Portal → 앱 선택 → Keys and tokens
📋 발급 항목 및 저장
  1. API Key & API Key Secret

    Consumer Keys 섹션 → Regenerate 클릭
    팝업에서 두 값 즉시 복사 (창 닫으면 재확인 불가)

  2. Access Token & Access Token Secret

    Authentication Tokens 섹션 → Access Token and Secret → Generate
    팝업에서 두 값 즉시 복사 (창 닫으면 재확인 불가)

  3. 안전한 곳에 임시 보관

    메모장에 아래 형식으로 저장 후 다음 단계에서 GitHub에 입력

📝 임시 메모 형식 (절대 공개 저장 금지) 
# X API 키 — GitHub Secrets에 등록 후 이 파일 삭제할 것

X_API_KEY        = 여기에_API_Key_붙여넣기
X_API_SECRET     = 여기에_API_Key_Secret_붙여넣기
X_ACCESS_TOKEN   = 여기에_Access_Token_붙여넣기
X_ACCESS_SECRET  = 여기에_Access_Token_Secret_붙여넣기
🚨
절대 코드에 직접 입력 금지
API 키를 .env.local, 코드, 커밋에 절대 넣지 마세요. 반드시 GitHub Secrets를 통해서만 관리

문제 해결

❌ 자주 발생하는 오류
  1. 403 Forbidden — 트윗 권한 없음

    X Developer Portal → 앱 설정 → App permissions가 Read and write인지 확인
    권한 변경 후 Access Token을 재생성해야 적용됨

  2. 401 Unauthorized — 인증 실패

    GitHub Secrets 값에 앞뒤 공백이 없는지 확인
    Secret을 삭제하고 재등록 시도

  3. API fetch 오류 — 운세 데이터 없음

    luckytoday.one 서버가 정상 작동 중인지 확인
    브라우저에서 직접 접속: luckytoday.one/api/horoscope/daily/aries

  4. GitHub Actions 미실행

    repository Settings → Actions → General → Allow all actions 활성화 확인

💻 로컬에서 수동 테스트 (DRY RUN) 
# .env.local에 X API 키 추가 후:
npm run post:horoscope:dry    # 트윗 내용만 출력 (포스팅 없음)
npm run post:horoscope        # 실제 포스팅
