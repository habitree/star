# 별자리 기초데이터 및 운세 콘텐츠 라이브러리

## 개요

이 디렉토리는 운세 생성 시스템의 콘텐츠 다양성을 개선하기 위한 **원천 데이터 저장소**입니다.

### 배경
- **기존 시스템**: ~275개 템플릿 (일반 75 + 원소별 180 + 조언 10 + 주간/월간 10)
- **문제점**: 결정론적 시드 + 제한된 풀 → 약 2주 내 반복 패턴 노출
- **목표**: 3,500+ 운세 소재 확보 → 약 12.7배 콘텐츠 다양성 향상

## 각 파일의 용도

### signs/ - 12별자리 마스터 레퍼런스

각 별자리 문서는 다음 섹션을 포함합니다:

| 섹션 | 내용 | 운세 활용 |
|------|------|----------|
| 기본 정보 | 기간, 심볼, 원소, 모달리티, 지배행성 | 기존 `zodiac-signs.ts` 보강 |
| 데칸 분류 | 3데칸별 부지배행성, 핵심성향, 키워드 10개 | 데칸별 운세 차별화 |
| 성격 특성 | 강점 10+, 약점 10+, 위기반응 패턴 | 종합운 문구 소재 |
| 사랑/연애 | 연애스타일, 솔로/커플 특성, 궁합 | 사랑운 문구 소재 |
| 직업/커리어 | 업무스타일, 적합직업 10+, 리더십 유형 | 직업운 문구 소재 |
| 건강 | 관장 신체부위, 취약영역, 추천활동 | 건강운 문구 소재 |
| 재물/금전 | 소비패턴, 투자성향, 주의점 | 금전운 문구 소재 |
| 상징 데이터 | 행운색상 5+, 보석 3+, 숫자, 꽃/동물 | 럭키 아이템 확장 |
| 신화/역사 | 그리스 신화, 동양 점성술 연관 | 콘텐츠 깊이 |
| 유명인 | 한국인 포함 15명+ | 페이지 콘텐츠 |
| **운세 문구 소재** | **5카테고리 × 3레벨 × 10개 = 150문구** | **직접 템플릿화** |

**별자리당 150문구 × 12별자리 = 1,800개 신규 소재**

## 데이터 → 템플릿 변환 파이프라인

```
┌─────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  1. 원천 데이터   │ ──→ │  2. 한국어 문구    │ ──→ │  3. 다국어 번역   │
│  (이 디렉토리)    │     │  추출/정제        │     │  (ko→en/zh/ja/es) │
└─────────────────┘     └──────────────────┘     └──────────────────┘
                                                           │
┌─────────────────┐     ┌──────────────────┐              │
│  5. 기존 시스템   │ ←── │  4. TypeScript    │ ←───────────┘
│  통합/배포       │     │  코드 생성        │
└─────────────────┘     └──────────────────┘
```

### Step 1: 원천 데이터 (현재 단계)
- `signs/*.md`: 별자리별 상세 정보 + 150개 운세 문구
- `reference/*.md`: 행성/달/데칸/계절 레퍼런스 문구
- `templates/phrase-library.md`: 조합용 표현 패턴

### Step 2: 한국어 문구 추출
- 마크다운 문서에서 운세 문구 섹션 파싱
- 중복 제거, 길이 검증 (30~80자)
- 카테고리/레벨별 분류 확인

### Step 3: 다국어 번역
- `LocalizedText` 포맷: `{ ko, en, zh, ja, es }`
- 각 언어의 자연스러운 표현 유지
- 문화적 맥락 반영

### Step 4: TypeScript 코드 생성
- `src/data/sign-templates.ts` (별자리별 전용 템플릿)
- `src/data/decan-templates.ts` (데칸별 템플릿)
- `src/data/transit-templates.ts` (행성 트랜짓 기반 템플릿)
- 기존 `HoroscopeTemplates` 인터페이스 준수

### Step 5: 시스템 통합
- `src/lib/horoscope-generator.ts` 에서 새 템플릿 풀 활용
- 결정론적 시드 시스템과 호환
- 기존 원소별 템플릿과 계층적 결합

## 기존 코드와의 관계

### 관련 소스 파일
```

├── horoscope-templates.ts    # 일반 템플릿 (75개)
```

### 데이터 일관성
- 별자리별 원소, 지배행성, 모달리티 등의 기본 정보는 `zodiac-signs.ts`와 일치하도록 작성
- 원소 분류 (Fire/Earth/Air/Water)는 `element-templates.ts`와 일치
- 행성-별자리 매핑은 `planet-influences.ts`의 `signToPlanet`과 일치

## 품질 기준

1. **문구 길이**: 30~80자 (한국어 기준)
2. **톤 분류**: high=격려/긍정, medium=균형/중립, low=부드러운 경고/건설적 조언
3. **카테고리 분류**: 종합운, 사랑운, 직업운, 건강운, 금전운 (5개)
4. **별자리 특성 반영**: 원소, 지배행성, 모달리티 특성이 문구에 반영
5. **실행 가능한 조언**: 단순 예측이 아닌, 행동할 수 있는 조언 포함
6. **문화적 적절성**: 한국 문화에 맞는 자연스러운 표현
7. **중복 배제**: 모든 문구가 고유한 표현

## 향후 활용 계획

1. **Phase 1**: 별자리별 전용 템플릿(`sign-templates.ts`) TypeScript 변환
2. **Phase 2**: 데칸별 차별화 시스템 구현
3. **Phase 3**: 계절/행성 기반 동적 문구 시스템
4. **Phase 4**: 조합형(composite) 템플릿 시스템으로 이론적 무한 조합 달성
